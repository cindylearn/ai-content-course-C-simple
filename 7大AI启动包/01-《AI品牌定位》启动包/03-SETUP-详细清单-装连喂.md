> 🔴 **两条底盘规则（跟 SKILL.md / 本包 README 一致，做这一步也守）：** ① **交互**：问用户任何选择题 → 一题一题、一题一张可点选项卡（每题≤4 个+「其他」）、**绝不一次全倒**。② **底层三层**：出的每样东西都套 说服（大师：**Blair Warren 为处境正名**/自我说服/LF8/AIDA·PAS/Cialdini/Duarte）+ 语言（NLP）+ 动机（6 大人性需求），**说得出「用了哪个大师 + 哪个 NLP + 戳哪个需求」，说不出＝重来。**

| **`ai-content-factory`**（本 skill 包） | 系统总图 + 生产线 + 协作框架 + 6 工具 skill + 交互教练 | ✅ 必须 |
| **`suma-copywriting`**（文案 skill） | 按品牌语气/分层框架写文案 | ✅ 强烈建议 |
| **`script-to-video-prompt`** | 脚本 →（9宫格选角锁人物）→ 每分镜 video prompt。<br>📄 `7大AI启动包/04-《AI口播短视频》启动包/VIDEO-PROMPT-SKILL.md`（**规则以同目录 README 为准**） | ✅ 视频要用 |

## ② 连什么 CONNECTOR（MCP / 集成）

| Connector | 用途 | 必须? |
|------|------|------|
| **Notion** | 读写内容矩阵数据库（立项/填文案/贴成品/改状态） | ✅ 必须 |

## ③ 喂什么 CONTEXT / 资料（地基 — 最关键，决定输出对不对）

我不是凭空指挥 Claude，是**先喂了一整套品牌地基**（每一类写成一个 memory / doc）：

**A. 品牌与生意**
- 生意资料：课程/产品、学费/定价、师资、目标学生
- 品牌 voice：使命/愿景/口号/核心信息/语气
- 品牌资产：**品牌色 hex、字体、logo、CTA 链接、平台矩阵**
- 人员档案：谁负责什么（Cindy/Jared…）
- 所有链接汇总：Google Sheets、Drive、GitHub、官网、WhatsApp

**B. 合规红线（护栏）**
- 合规规则：哪些词不能用、哪些证照/logo 有法律规定、不写价格/免费/具体名额

**C. 策略与框架**
- 营销 SOP：销售话术、Open Day 框架、考核 Q&A
- 内容脚本库：**广告脚本角度、hook 模板、内容操作系统、Meta Ads 策略**
- 核心定位：**「卖方法论不卖工具」**——缺的是「怎样用 AI 有效做出东西」，不是工具本身

**D. 产品/课程细节**
- VSL 定稿、LP 架构与内容、启动包 bonus、客服 KB、AI 角色人设

**E. 参照范本（学习框架 — 见 ④）**

> 关键：这些不是一次喂完的，是**边做边补**，踩一个坑记一条（「记得更新 sop 和 md」）。

## ④ 我给过的「学习框架」（参照范本）

这套内容质量高，是因为我给了 Claude **现成的范本去学**，不是让它瞎写：

1. **UK Diploma 文案手法** — 「学习我另一个 project【UK Diploma】的文案写作手法」+ 丢过去 **3 个 Google Doc 链接**（ToFu / MoFu / BoFu 三层各一份范例）。→ Claude 学会了分层框架。
2. **Hook 谱 / 内容角度库** — 痛点直击·对比反差·趋势警醒·行业揭秘·实操演示·学员见证·算账ROI·招生CTA（从内容脚本库来）。
3. **内容操作系统 / Meta Ads 策略** — 谁的打法、怎么分层投放。
4. **参照锁定法** — 做出满意的例子后：「用这个做例子，我要每个都这样」/「文案的 part 用这种画面」。

> 学习框架 = 给 AI「对的样子」，比给一堆规则更有效。

- **Logo 文件（2 张）**：浅底版 + 深底版（给 PIL 自适应合成用）。放在固定路径。

- 生产/合成/拼接要的本地依赖：`pip install pillow imageio-ffmpeg`（+ 归档用 `rclone`）。

## ⑥ 我「喂」的方式（step by step，可复用的动作）

1. **先建持久 memory**：把①③④写成 md 存进记忆系统 → Claude 每次先读地基。
2. **给参照物**：「学习我另一个 project 的手法」+ 丢链接/Google Doc。
3. **用截图喂现状**：Drive 结构、Notion 表、图裂报错、丑海报——**直接甩截图**，不用嘴描述。
4. **用例子锁定方向**：指一个满意的成品说「就这样，全部照这个」。
5. **看成品→挑具体的刺→纠正→记成规则**：不满意就指出到可执行（太虚化/像 aunty/有乱码），并「更新 sop 和 md」。
6. **决策闸门**：生产前「先不生产 / 确认后才生成」；状态标 Proceed 才花钱。

## 🔴 常见问题：Claude 问我问题时，没有「可以点的选项卡」怎么办？

教练模式本该用**可点击的选项卡**让你点选（不用打字）。如果你看到的只是文字「1. 2. 3.」，按下面排查：

| 你用什么开的 Claude | 有没有选项卡 | 怎么办 |
|---|---|---|
| 🌐 **网页版 claude.ai**（浏览器）| ❌ **本来就没有** | 换用电脑版 Claude（App / 命令行）才有选项卡 |
| 💻 **电脑版 App / 命令行**，版本**新** | ✅ 有 | 正常 |
| 💻 **电脑版**，版本**旧** | ❌ 没有（功能还没到）| **更新到最新版**，再重开 |

**排查三步：**
1. 问 Claude 一句：**「我的 Claude 是什么版本？为什么问我问题时没有可以点的选项卡？」** —— 它会告诉你版本、要不要更新。
2. **旧就更新**到最新版（App 里检查更新 / 命令行重装），重开。
3. 更新后**重新拉一次这个 skill 的 GitHub**（装到最新版），再调用。

> ⚠️ 就算一切正常，**每题选项卡最多 4 个**（这是工具的硬限制）。选项超过 4 个的题（如「气质」8 种），Claude 只会把最相关的 3–4 个做成选项卡，其余用「其他」让你打字补充 —— 这是正常的，不是坏了。
