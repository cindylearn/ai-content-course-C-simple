## 步骤 3 · 数据驱动的优化策略

### 1. A/B 测试（先测再放量）
- **明确一个变量**：一次只测一个，否则分不清是谁的功劳。常测变量：
  - 文案：**标题 / 头图 / 首行钩子 / CTA 文案**
  - 图文：**封面 / 主视觉 / 排版顺序**
  - 视频：**前 3 秒 Hook / 缩略图 / CTA 按钮颜色**
- **定评估标准**：先说清用什么指标判胜负（CTR？CVR？完播率？留资成本？）+ 需要多少样本/多长时间才算数。
- 赢的版本 → 变成新基线，再测下一个变量。

### 2. 同类群组分析（Cohort）+ 受众细分
- 按「进来的时间/来源/首次行为」把用户分成 **cohort**，看**留存曲线**随时间怎么走 —— 比看总数更能发现「哪批人留得住、哪批漏得快」。
- 结合 **RFM 分层**（最近消费 / 频率 / 金额）给高净值用户单独的定向和内容。

### 3. 跨渠道归因（别被单一归因骗）
- 一个转化往往经过多次触点（看到短视频 → 搜品牌 → 点广告 → 成交）。只认「最后一次点击」会**低估内容/社媒的真实价值**。
- 用多触点归因看清**每个渠道的真实贡献**，再决定预算往哪加。

### 4. 自动化 + 智能出价
- 把重复的生产/发布/追踪交给脚本和自动化（见 `SKILL.md` 自动化编排）。
- 投放用平台的**机器学习出价**做个性化触达 + 预算效率最大化；人只定策略和上限。

> 🔧 落地：把每条内容的表现（CTR/CVR/花费）回填进 Notion 内容矩阵，让「哪个受众×哪层×哪个钩子」表现好一目了然 → 下一批照赢的方向放量。

## 步骤 4 · 专业汇报与沟通（也是所有输出的格式规矩）

- **翻译术语**：把 AARRR、归因、Cohort 这些词，转成用户听得懂的商业洞察（「你的问题不在获客，在留存——第 2 周就掉一半人」）。
- **排版必须清晰**：**粗体** + 项目符号 + 标题结构，让每一步优化建议一眼看清。
- **结尾抛一个启发性问题**：引导用户想下一步 / 补上下文。例：
  > 「你现在最大的瓶颈是**拉新不够**，还是**来了留不住**？告诉我，我帮你把预算和内容重心调到对的地方。」
