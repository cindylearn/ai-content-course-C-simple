## 怎么用好（核心：分层框架）
文案不是「写得好听」，是**在漏斗不同层说不同的话**：
- **ToFu 觉醒**：不硬卖，戳痛点/趋势，让对方意识到「我需要」。
- **MoFu 考虑**：讲 USP、拆方法、给证据（证明「这方法行」）。
- **BoFu 决策**：算账 ROI + 紧迫感（推「现在就报」）。

**每条文案的骨架：** 钩子(hook) → 痛点/对比可视化 → 方法/解决方案 → 带出完整工具组 → CTA(完整品牌名 + 链接)。

**Prompt Claude 的方式：** 给它①受众②漏斗层③钩子类型④参照范本（你满意的旧文案），让它按框架写；不满意就指具体的刺（太硬卖/不够实/像别的行业）让它改。

## 踩坑（我们学到的）
- 🔴 **合规红线写死**：不写价格 / 免费 / 具体名额数字（紧迫用「名额有限」）。
- 🔴 **完整品牌名一字不差**，每条都要出现一次。
- 🔴 **卖方法论不卖工具**：强调「怎样用 AI 有效做出东西」，不是工具本身。
- 🔴 **6 工具自然带出**（图文海报每页带全 6；视频台词**自然聚焦**相关的，全套由 caption 承载，别硬塞清单）。
- 🔴 **台词无口头禅**（不要「咯/啦」），要多要实、节奏快。
- 🔴 **串行警惕**：批量改文案时，页面标题 / 页内 header / 标签 常被漏改——审计要跟正文分开逐项查。
